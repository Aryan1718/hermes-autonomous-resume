# Change Log

Use this file to track meaningful profile updates for the current candidate.

| Date | Change | Reason |
|---|---|---|
| YYYY-MM-DD | Initial profile created | Repository setup for current candidate |
| YYYY-MM-DD | Updated role history | Candidate added or corrected work experience |
| YYYY-MM-DD | Added provable skills/signals | New evidence from projects, OSS, or work |
| YYYY-MM-DD | Updated hard constraints | Geography, authorization, or seniority rules changed |
| YYYY-MM-DD | Rebalanced target roles | Candidate narrowed or expanded job-search focus |

## Logging Rules

- Add one row per meaningful profile-editing session.
- Record what changed, not every line touched.
- Keep the reason tied to candidate facts or strategy changes.
- Do not preserve stale entries from a previous candidate when reusing this repository.
