# Question Groups

Ask for the minimum fields required to make `candidate-profile` usable. Keep the interview batched and concise.

## Group 1 - Core Identity And Search Scope

Collect:
- candidate name
- current education status, if relevant
- current role title
- current company
- most recent prior role, if important
- target country or countries
- location scope
- company stage scope, if the candidate has one

## Group 2 - Role Fit And Seniority

Collect:
- in-scope role titles
- in-scope role types
- explicit out-of-scope role types
- target seniority
- years of relevant experience
- whether reasonable stretch roles should still pass

## Group 3 - Work Authorization And Hard Constraints

Collect:
- current work authorization status
- future sponsorship need
- any geography restrictions
- any industry restrictions
- any hard disqualifiers beyond location, authorization, seniority, and role-family mismatch

## Group 4 - Provable Skills

Collect:
- 4-8 always-provable skills or experiences
- best evidence source for each one
- 2-5 provable-with-framing skills
- a short note for each framing-only item

Reject vague answers like "good at backend" unless the user ties them to real work, projects, OSS, or coursework.

## Group 5 - Signals And Direction

Collect:
- 2-5 strongest signals
- why each one is especially credible
- 2-4 career direction fits, in ranked order
- preferred signals grouped by area

## Group 6 - Optional Runtime Setup

Ask this only if the user wants the repository itself personalized beyond `candidate-profile`.

Collect:
- profile slug
- pool directory name
- resumes directory name
- dashboard base URL
- dashboard API key env var name

If the user does not want infra setup yet, skip this group entirely.
